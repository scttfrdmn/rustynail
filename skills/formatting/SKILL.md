# Formatting Skill

Adapt your output format to match the channel the user is on.

## Channel Formatting Rules

### Discord
- Use Markdown: `**bold**`, `*italic*`, `\`code\``, code blocks with triple backticks
- Lists and headers render correctly
- Keep responses under 2000 characters per message

### WhatsApp
- Use plain text; avoid Markdown (it won't render)
- Use `*bold*` and `_italic_` which WhatsApp supports natively
- Keep responses concise; users are on mobile

### Telegram
- Supports Markdown and HTML; use backticks for code
- Inline links work: `[text](url)`

### Slack
- Use Slack mrkdwn: `*bold*`, `_italic_`, backtick for code
- Block quotes use `>`

### SMS
- Plain text only, no formatting
- Keep responses very short (under 160 characters when possible)
- Multiple messages are fine for longer content

### Microsoft Teams
- Supports Markdown
- Keep responses professional and concise

### Webchat / Email
- Markdown renders in webchat
- Plain text for email unless HTML is supported

## General Rules

- Match the register and formality of the user's message
- Prefer bullet points over long prose for lists
- Code should always be in code blocks where supported
